# Quorima - sessie-handoff 2026-06-13

Voor een verse sessie die in deze repo (`Quorima/`) verder bouwt.

## Wat Quorima is

Een agentic C-level board (SaaS) voor de holding Sirrapa Group Holding B.V. (100% Armand). Een Chief of Staff orchestreert vier C-level agents (CEO, CFO, COO, CMO) over drie werkmaatschappijen. Alle redeneerwerk via platform.claude.com (Claude API + Agent SDK). Tools volgen `domain.action`. Escalaties lopen altijd via Chief of Staff -> `escalate_to_human()`.

Werkmaatschappijen, must-watch KPIs en primaire agent:

- Sirrapa (ICT) B.V. (Twinfield 21005, template ICT-Solo-Freelance, CMO+CEO): klant-concentratie %, effectieve maandomzet (target 18.000 euro), pipeline-diversificatie. Risico: 100% klant-concentratie.
- Sirrapa Vastgoed B.V. (Twinfield 21007, template Property-Income-Geared, CFO+CEO): DSCR, NOI/mnd, refi-runway. Acuut onder druk.
- Sirrapa Property Group Ltd (Xero, GBP, template Property-C2R-Acquisition-Phase, CFO+COO): qualified pipeline GDV, time-to-LOI, capital deployment. Acquisitie-fase, nog geen project.

## Wat al bestaat in de repo

- `agent_prompts/` - system prompts voor Chief of Staff + CEO/CFO/COO/CMO.
- `kpis/` - KPI-definities per werkmaatschappij + UK acquisition-phase.
- `quorima-mvp/` - werkende CFO daily-flash (TypeScript, Twinfield SOAP + Claude Opus). `npm run flash:dry-run` werkt zonder keys op mock data. Architectuur: Ports/adapters (AccountingPort), domain (kpi.ts/escalation.ts), agents/cfo.ts, digest/render.ts.
- `wizard/sirrapa_tenant_config.example.json` - tenant-config (entiteiten, connectors, cadans, escalatie-routing).
- `connectors/` - Twinfield/Xero setup-notities.
- `branding/` - brand guide + logo-assets (Quorima purple #6b46c1 / navy, Lato, Round Table-mark).
- `strategy/` - ICT-groei 5k->20k MRR, UK C2R case study.

## Cross-link naar het Hermes/CFO-deel — twee CFO-lagen

Er zijn **twee CFO-lagen**, complementair, niet dubbel:

- **Hermes-CFO (draait nu):** CFO-*admin/operatie* = de Gmail factuur-pipeline. Runtime Hermes (Linux, cron, always-on); Gmail-acties door agent Freya, redeneerlaag Gerrit (ATLAS). Triëert/routeert facturen (incasso→Basecone, te-betalen→monitor, bon→archive), approval-gated (enige autonome externe verzending = forward naar bekend Basecone-adres, spend €0). Geautoriseerd op `armand.parris@sirrapa.com`. Docs: MyBrain `03. Areas/financial-control/gmail-invoice-pipeline-unified-spec.md` (leidend) + `gmail-pipeline-handoff-2026-06-13.md`.
- **Quorima-CFO (deze repo, MVP):** CFO-*board/analyse* = de daily flash. Leest Twinfield grootboek → DSCR/NOI/refi + escalaties.

**Hoe ze koppelen:** Hermes duwt facturen de boeken in (Basecone→Twinfield); Quorima leest ze eruit voor KPI's — zelfde pijplijn, tegengestelde richting. De Basecone-adressen `kubusalkmaar.<office>@mailtobasecone.com` gebruiken de Twinfield office-codes uit deze tenant-config (21005 ICT, 21006 Holding, 21007 Vastgoed); SPG/UK boekt op Xero. De openstaande facturen op het dashboard komen nu uit een echte feed (`dashboard/data/invoice-overview.json`), gevoed door deze Hermes-pipeline — zie `dashboard/README.md` ("Factuur-feed (Hermes)").

**Gedeeld open punt:** canoniek domein — `sirrapa.com` (operationeel, Hermes draait hierop) vs `sirrapagroup.com` (merk/escalatie-adres). Staat in beide projecten open (zie ook open punt 5 hieronder).
